# 모두의 ETF 요구사항 정리

이 문서는 현재 프로젝트에서 논의하고 반영한 기능/설계 요구사항을 정리한다.

문서 구성은 아래 4단으로 읽으면 된다.

- `1 ~ 11`: 원본 요구사항
- `12 ~ 25`: 원본 이후 실제 작업 과정에서 확정된 변경 이력
- `26 ~ 36`: 기존 스레드 이후 추가/수정된 후속 반영 사항
- `37 ~ 41`: 이 스레드에서 추가/수정된 후속 반영 사항
- `최신 통합 기준`: 현재 시점 기준 최신 통합 기준
- `빠른 참조용 구조화 요약`: 다음 스레드/다음 작업자가 빠르게 참고할 수 있는 구조화 요약

권장 읽기 순서:

- 원래 의도를 보려면 `1 ~ 11`
- 무엇이 바뀌었는지 추적하려면 `12 ~ 25`
- 직전 스레드 반영분만 빠르게 보려면 `26 ~ 36`
- 이번 스레드 반영분만 빠르게 보려면 `37 ~ 41`
- 지금 기준으로 바로 구현하려면 `최신 통합 기준`
- 빠르게 업무를 이어가려면 `빠른 참조용 구조화 요약`

## 1. 제품 개요

- 서비스명은 `모두의 ETF`
- 사용자는 추천 포트폴리오를 먼저 확인하고, 로그인 후 자신의 포트폴리오를 이어서 관리한다.
- 서비스는 단순 종목 추천이 아니라 다음 두 축을 함께 제공한다.
  - 추천 포트폴리오 보기
  - 내 포트 관리하기

## 2. 데이터/분석 구조

### 2.1 핵심 팩터

- 추천 점수는 아래 3팩터를 사용한다.
  - 성장
  - 가치(PER)
  - ROE

### 2.2 성향별 가중치

- 안정형: 성장 20 / 가치 50 / ROE 30
- 밸런스형: 성장 40 / 가치 40 / ROE 20
- 공격형: 성장 60 / 가치 30 / ROE 10

### 2.3 연도 로직

- 기준 연도는 현재 연도 기준으로 자동 계산한다.
- 성장/PER 시작점:
  - `기준 연도 - 1` 확정 실적
- 성장/PER 끝점:
  - `기준 연도 + 2` 전망치 우선
  - 없으면 `기준 연도 + 1` 전망치
- ROE:
  - `기준 연도 - 1` 연간 확정값

### 2.4 제외 종목 처리

- `기준 연도 + 2` 전망치가 있으면 사용
- 없으면 `기준 연도 + 1` 전망치 사용
- 둘 다 없으면 제외
- 금융/증권 업종은 업종 특성상 일반 제조업과 다른 항목명을 사용할 수 있으므로 별도 대응

### 2.5 비중 산식

- 추천 포트폴리오 비중은 단순 점수 비례가 아니라 `점수 제곱 비례`를 사용한다.
- 상위 종목일수록 비중 차이가 더 분명하게 드러나도록 한다.

## 3. 종목 선택/유니버스

### 3.1 기본 원칙

- 종목은 버튼형 카테고리 선택 + 개별 검색/추가/제외 방식으로 고른다.
- ETF는 선택 대상에서 제외한다.
- 선택 후보는 시가총액 순으로 보여준다.

### 3.2 기본 관심 종목

- 기본 관심 종목 세트가 존재한다.
- 논의 과정에서 아래 종목이 기본 관심 종목에 추가되었다.
  - 우리금융지주
  - 메리츠금융지주
  - SK텔레콤
  - KT
  - 대한항공

### 3.3 시장 카테고리

- 시장 카테고리별 후보를 빠르게 고를 수 있어야 한다.
- 이후 개별 종목을 검색해서 조정할 수 있어야 한다.

## 4. 추천 포트폴리오 화면

### 4.1 목적

- 투자 가능 금액과 투자 성향을 기준으로 산출한 추천 포트폴리오를 보여준다.

### 4.2 표시 항목

- 도넛 차트 + 비중 범례
- 각 종목별 상세 카드
- 종목 상세에는 아래 값을 보여준다.
  - 성장률
  - PER
  - ROE
  - 총 점수
- 성장률과 PER은 각각 실제 대표값 하나로 정리해서 보여준다.

### 4.3 종목 성향 분류

- 성장 점수와 가치 점수를 기준으로 분류한다.
- 분류명:
  - 성장·가치 동시 충족
  - 성장형
  - 가치형
  - 균형 관찰형
- 분류 기준:
  - 성장 점수 60 이상 여부
  - 가치 점수 60 이상 여부

### 4.4 인사이트 시각화

- 성장 vs 가치 맵
- 성장/가치/ROE 비교표
- 선택 종목 기준으로 점수와 위치를 비교할 수 있어야 한다.

## 5. 내 포트 관리하기

### 5.1 목표

- 기존 보유 포트폴리오를 저장하고
- 이후 매수/매도 거래를 추가하여
- 변경 전후를 비교하고
- 최종적으로 현재 포트폴리오를 갱신한다.

### 5.2 저장 방식

- 로그인 후 사용자별 포트폴리오 저장
- 여러 기기에서 이어서 관리 가능

### 5.3 포트폴리오 슬롯

- 사용자는 여러 포트폴리오를 저장할 수 있다.
- 지원 기능:
  - 신규 포트폴리오 만들기
  - 복사하기
  - 이름 변경하기
  - 삭제하기
- 동일한 이름의 포트폴리오는 만들 수 없다.
- 신규 포트폴리오는 기본값으로 생성한다.
  - 밸런스형
  - 종목 0개
  - 총평가액 0원

### 5.4 현재 보유 포트폴리오 표시

- 증권사 잔고 화면처럼 보여준다.
- 상단 요약 카드:
  - 총매입금액
  - 총평가금액
  - 총평가손익
  - 총수익률
  - 예수금잔액
  - 실현손익
- 종목별 행:
  - 종목명
  - 보유 주수
  - 비중
  - 평균 매입 단가
  - 현재가
  - 수익률
  - 수익금
- 수익률/평가손익은 양수/음수에 따라 색상 구분

## 6. 거래내역과 장바구니

### 6.1 거래내역 입력 방식

- 기존 보유 종목 목록을 다시 나열하는 방식이 아니라
- 검색 + 시가총액 상위 카드에서 종목을 골라 거래를 추가한다.
- 검색 결과는 실제로 동작해야 한다.
- 상단에는 시가총액 상위 종목 카드가 보인다.

### 6.2 단일 종목 거래 입력

- 종목 카드를 클릭하면 해당 종목의 거래 입력 섹션이 열린다.
- 입력 항목:
  - 거래 주수
  - 거래 단가
- 기본값:
  - 주수는 빈칸
  - 단가는 현재가

### 6.3 빠른 입력 카드

- 거래 주수 카드:
  - +5주
  - +10주
  - 1/2
- 거래 단가 카드:
  - 감소 3개
  - 증가 3개
- 표시 문구는 `호가×5`처럼 내부 로직 기준이 아니라 실제 변동 금액으로 보여준다.
- 호가 단위는 가격대가 바뀌면 같이 바뀌어야 한다.

### 6.4 입력 UX

- 수량/단가 입력은 `왼쪽 - / 가운데 입력 / 오른쪽 +` 구조
- 브라우저 기본 위아래 스피너는 숨긴다.
- 입력칸은 비워둘 수 있어야 한다.

### 6.5 장바구니

- 매수 담기 / 매도 담기를 누르면 아래 장바구니에 쌓인다.
- 매수 담기는 빨강, 매도 담기는 파랑으로 표현
- 장바구니는 별도 제목으로 구분한다.
- 장바구니 항목에는 아래를 한 줄로 배치한다.
  - 종목
  - 거래 주수
  - 거래 단가
  - 예상 금액
  - 제거
- 제거 버튼은 예상 금액 오른쪽에 위치
- 장바구니에서 수량/단가 수정 가능
- 장바구니에서 제거하면 아래 변경 내역도 즉시 반영

## 7. 포트폴리오 업데이트 비교

### 7.1 목적

- 기존 포트폴리오 대비 변경된 내역을 확인하고 새로운 포트폴리오로 업데이트한다.

### 7.2 비교 방식

- 기존 저장 포트폴리오 vs 업데이트 예정 포트폴리오를 비교
- 위쪽에는 파이차트로 비중 변화 비교
- 아래 카드에서는 `주수 변화가 있는 종목만` 표시

### 7.3 카드 정렬/표시

- 정렬은 변경 후 비중 높은 순
- 카드에는 적어도 아래 내용을 포함한다.
  - 기존 주수
  - 변경 후 주수
  - 주수 변화
  - 비중 변화
- 주수 증가: 파랑
- 주수 감소: 빨강

### 7.4 확정

- 거래를 장바구니에 담는 즉시 포트폴리오에 반영하지 않는다.
- 비교 결과를 확인한 뒤, 사용자가 명시적으로 확정할 때 저장한다.
- 확정 시 저장 완료 안내 메시지를 보여준다.

## 8. 리밸런싱 보기

### 8.1 목적

- 추천 포트폴리오와 기존 포트의 차이를 읽기 전용으로 확인

### 8.2 실행 순서

- 전량 매도/비중 축소를 먼저 정리
- 이후 신규 편입/비중 확대 진행
- 카드 안에서는 주수 변화와 비중 변화를 함께 보여준다.

## 9. 인증과 저장 인프라

### 9.1 현재 방향

- 로그인 없이:
  - 추천 포트폴리오 보기 가능
- 로그인 후:
  - 내 포트폴리오 저장/관리 가능

### 9.2 인증 방식

- 이메일 링크 인증은 rate limit 문제로 제외
- Google OAuth 사용

### 9.3 저장소

- Supabase 사용
- MongoDB보다 관계형 구조가 더 적합하다고 판단
- `profiles` 테이블과 사용자별 RLS 정책 사용

## 10. 자산배분 확장 계획

이 부분은 아직 차기 단계 설계이며, 현재 구현 범위 밖이다.

### 10.1 상위 자산군

- 현금/예적금/채권
- 금/달러
- 가상자산
- 주식

### 10.2 주식 세분화

- 국내주식
- 해외주식

### 10.3 자산배분 성향 프리셋

- 안정형
  - 현금/예적금/채권 50
  - 금/달러 20
  - 가상자산 0
  - 주식 30
- 밸런스형
  - 현금/채권 30
  - 금 10
  - 가상자산 10
  - 주식 50
- 공격형
  - 현금/채권 20
  - 가상자산 20
  - 주식 60

### 10.4 환율 기반 국내/해외 비중 조절

- 1300원 이하: 해외 100 / 국내 0
- 1400원: 해외 50 / 국내 50
- 1500원 이상: 해외 0 / 국내 100
- 그 사이는 선형 보간

## 11. 후속 확장 후보

### 11.1 목표가 모멘텀

- 절대 목표주가보다는 최근 리포트의 상향/하향 방향성 활용
- 지표 아이디어:
  - `(상향 건수 - 하향 건수) / (상향 건수 + 하향 건수)`
- 데이터 수집 난이도가 높아 후속 과제로 분류

### 11.2 모바일 대응

- 모바일에서는 데스크톱 레이아웃 축소가 아니라
- 전용 카드형/세로형 구조로 재구성 필요

## 12. 로그인 전/후 화면 분리

- 로그인 전에는 `추천 포트폴리오 보기` 경험만 노출한다.
- 로그인 후에만 `내 포트 관리하기`를 노출한다.
- 로그인 전 추천 화면에서는 실행 순서까지만 보여주고, 그 아래의 실제 반영 단계는 숨긴다.
  - 추가 현금 납입
  - 리밸런싱 장바구니
  - 리밸런싱 전후 비교
  - 확정 버튼
- 로그인 후에는 랜딩 상단 CTA에서 `구글로 로그인하기` 대신 `로그아웃`을 보여준다.
- 아래 `내 포트 관리하기` 섹션에서는 별도 로그아웃 버튼을 두지 않는다.

## 13. 비로그인 기본 추천 조건

- 로그인하지 않은 상태의 기본 추천 포트폴리오 조건은 아래와 같다.
  - 종목 카테고리: 기본 관심 종목
  - 투자 성향: 밸런스형
  - 투자 금액: 1,000만원
- 비로그인 상태에서 계산되는 추천 예시도 실제로 1,000만원 기준이어야 한다.

## 14. 로그아웃 시 상태 초기화

- 로그아웃하면 새로고침 없이도 작업 상태를 즉시 초기화해야 한다.
- 즉시 초기화 대상:
  - 보유 종목
  - 저장된 포트폴리오 선택 상태
  - 거래 장바구니
  - 리밸런싱 장바구니
  - 현금 조정 상태
  - 비교/확정 대기 상태
- 로그아웃 이후에는 추천 전용 기본 화면으로 즉시 돌아가야 한다.

## 15. 랜딩 페이지 타이포/메시지

- 랜딩 상단 브랜드 `모두의 ETF`는 메인 설명 문구보다 더 크게 보여야 한다.
- 메인 설명 문구는 아래를 사용한다.
  - `추천 포트폴리오를 확인하고,`
  - `필요한 포트폴리오 변화를 자동으로 관리하세요.`
- 브랜드명은 강조, 설명 문구는 한 단계 낮은 위계로 정리한다.

## 16. 내 포트 관리하기 흐름

- `내 포트 관리하기`는 아래 순서의 흐름을 유지한다.
  - 현재 보유 포트폴리오 확인
  - 거래 입력
  - 거래 장바구니 적재
  - 현금 추가/인출 반영
  - 포트폴리오 업데이트 비교
  - 최종 확정
- 거래를 장바구니에 담는 즉시 저장하지 않는다.
- 비교 화면에서 변경 내용을 확인한 뒤 사용자가 확정할 때만 저장한다.

## 17. 추천 포트폴리오 리밸런싱 흐름

- 리밸런싱 기능도 `내 포트 관리하기`와 유사한 흐름으로 통일한다.
  - 실행 순서에서 선택
  - 리밸런싱 장바구니 적재
  - 현금 추가/인출 반영
  - 리밸런싱 전후 포트폴리오 비교
  - 최종 확정
- 리밸런싱 장바구니 바로 아래에서 즉시 반영하지 않는다.
- 확정 버튼은 비교 섹션 아래에 위치해야 한다.

## 18. 현금 추가/인출의 위치와 의미

- 현금 추가/인출은 추천 리밸런싱 쪽뿐 아니라 `내 포트 관리하기`에도 있어야 한다.
- 현금 변동은 단순 참고 정보가 아니라 실제 포트 변경 항목으로 취급한다.
- 현금 조정도 장바구니에 담고, 종목 거래와 함께 한 번에 확정한다.
- 비교 화면에서는 종목 변화 카드와 같은 레벨의 변경 항목으로 보여야 한다.

## 19. 장바구니 용어와 색 규칙

- 장바구니에서 매수/매도는 용어와 색으로 모두 구분해야 한다.
- 단가 표기:
  - 매수: `매입 단가`
  - 매도: `매도 단가`
- 행 색상:
  - 매수 행: 붉은 계열
  - 매도 행: 푸른 계열
- 종목명 아래 보조 설명 텍스트도 같은 색 규칙을 따라야 한다.

## 20. 실행 순서 / 변화 카드 색 규칙

- 이번 스레드에서 색 의미는 아래처럼 통일했다.
  - 양수 / 매수 / 신규 편입 / 비중 확대 = 빨강
  - 음수 / 매도 / 전량 매도 / 비중 축소 = 파랑
- 이 규칙은 아래 요소에 공통 적용한다.
  - 실행 순서 카드
  - 변화 배지
  - 포트폴리오 업데이트 비교
  - 리밸런싱 비교
  - 장바구니
- 라이트/다크 모드 모두 같은 의미 체계를 유지해야 한다.

## 21. 빠른 단가 조정 색 양식

- 빠른 단가 조정은 아래 색 구성을 기준으로 사용한다.
  - `단가 내리기` = 붉은 계열
  - `단가 올리기` = 푸른 계열
- 장바구니 색감도 이 빠른 단가 조정 패널과 같은 시각 언어를 따라야 한다.

## 22. 도넛 차트 옆 범례 정렬

- 추천 포트폴리오/업데이트 비교/리밸런싱 비교에서
- 도넛 차트 옆 범례는 항상 `비중 높은 순`으로 정렬한다.
- 종목 비교 카드 또한 가능하면 `반영 후 비중 높은 순`으로 정렬한다.

## 23. 포트폴리오 비교 카드 구조

- 기존의 세로형 변화 카드는 높이가 너무 커서 스캔성이 떨어졌으므로,
- 장바구니처럼 한 줄형 비교 카드 구조로 정리한다.
- 카드에는 중복 없이 아래 정보만 한 줄 구조로 정리한다.
  - 항목명
  - 기존 값
  - 변경 후 값
  - 변화 값
- 현금 변화도 동일한 패턴으로 표현한다.

## 24. 현재 보유 포트폴리오 / 실행 순서 UI 조정 기준

- `현재 보유 포트폴리오` 카드
  - 종목 카드 간 간격은 너무 붙지 않게 하되, 과도하게 넓지도 않게 조정
  - 카드 하단 메타 정보(매입총액/평가총액/평가손익/수익률)는 잘리지 않게 충분한 하단 여유 확보
  - 전체 높이는 실행 순서와 균형감 있게 맞춘다.
- `실행 순서` 카드
  - 카드 하단 3개 정보 박스는 지나치게 높지 않게 조정
  - 내부 3개 박스를 줄였다면 바깥 카드 높이도 같이 맞춰서 과하게 커 보이지 않게 조정

## 25. 저장/검증 운영 규칙

- Safari에서 개발 서버 기준으로 스타일 반영이 꼬인 경험이 있었으므로,
- 작업 마무리 시에는 아래 기준으로 확인한다.
  - `next build`
  - 기존 3000 포트 서버 종료
  - `next start`
  - `http://127.0.0.1:3000` 실제 응답 확인
- 완료 기준은 `코드 수정 완료`가 아니라 `최신 빌드가 3000 포트에서 정상 오픈 확인 완료`이다.

## 26. 투자 성향/환율 기반 자산배분과 국내주식 추천 구조 고도화

- 추천 포트폴리오는 먼저 상위 자산군 4개를 배분한 뒤, 그중 국내주식에 배정된 금액으로 생성한다.
  - 현금/예적금/채권
  - 금
  - 가상자산
  - 주식
- 상위 자산군 비중은 투자 성향에 따라 달라진다.
  - 안정형: 현금/채권 50 / 금 20 / 가상자산 0 / 주식 30
  - 밸런스형: 현금/채권 30 / 금 10 / 가상자산 10 / 주식 50
  - 공격형: 현금/채권 20 / 금 0 / 가상자산 20 / 주식 60
- 주식과 현금은 환율을 반영하여 아래처럼 세분화한다.
  - 주식: 한국주식 / 미국주식
  - 현금: 원화 현금 / 달러 현금
- 환율 구간은 아래 해석을 따른다.
  - 1,300원 이하: 달러 약세 구간
  - 1,400원: 중립 구간
  - 1,500원 이상: 달러 강세 구간
- 국내주식 추천 점수는 성장 / 가치(PER) / ROE를 기준으로 계산하고, 상위 10개 종목을 추천한다.
- 추천 포트폴리오 비중은 점수 제곱 비례로 계산한다.
- 종목 카테고리 선택 영역은 `국내주식 추천 포트폴리오` 섹션 아래에 둔다.

## 27. 추천 포트폴리오 정수 주수 처리 규칙 변경

- 추천 포트폴리오의 이상적인 주수는 먼저 `floor` 처리한다.
- 이후 남은 현금은 “소수점 나머지가 큰 순”이 아니라 “추천 상위 순서”대로 1주씩 반복 배분한다.
- 즉 추천 1위부터 차례대로 한 주씩 더 살 수 있으면 추가하고, 다시 처음부터 반복하는 방식이다.
- 목표는 정수 주수 강제를 유지하면서 추천 우선순위를 실제 실행 순서에 더 직접 반영하는 것이다.

## 28. 내 포트 관리하기 탭 유지 / 상태 복원 규칙

- 로그인 직후 첫 진입에서는 `내 포트 관리하기` 탭으로 이동하는 현재 동작을 유지한다.
- 다만 그 이후에는 새로고침이나 인증 상태 재확인 시에도 사용자가 마지막에 보던 탭을 유지한다.
- 같은 맥락으로 스크롤 위치도 가능한 한 유지/복원한다.
- 즉 `처음 로그인한 직후만 관리 탭 자동 진입`, 이후는 사용자가 보던 위치 유지가 기준이다.

## 29. 내 포트 관리하기 요약 카드 / 표 표시 규칙

- 상단 요약 카드는 2 x 4 구조를 사용한다.
- 1행:
  - 총 매입액
  - 총 평가액
  - 총 평가손익
  - 총 실현손익
- 2행:
  - 원금
  - 잔여 현금
  - 현재 수익금
  - 현재 수익률
- 계산 규칙:
  - 현재 수익금 = 총 평가손익 + 총 실현손익
  - 현재 수익률 = 원금 대비 현재 수익금의 비율
- 전일대비 등락률은 전체 요약에서 따로 보여주지 않는다.
- 대신 종목별 표에 `전일 대비` 컬럼을 두고 각 자산의 당일 등락률을 보여준다.

## 30. 자산배분 가이드 / 현재 내 자산 비중 시각화 규칙

- `4개 자산군 비중` 도넛 중앙은 종목 수가 아니라 총 평가액을 보여준다.
- 도넛 옆 상세 범례는 비중뿐 아니라 각 자산군의 평가액도 함께 보여준다.
- `현재 내 자산 비중` 상세설명은 차이 큰 순이 아니라 왼쪽 범례 순서 그대로 고정한다.
  - 현금/예적금/채권
  - 금
  - 가상자산
  - 주식
- 상세설명은 긴 문장형이 아니라 `현재 vs 가이드`를 시각적으로 비교하는 미니 바 차트 + 차이값 구조를 우선한다.
- 자산군과 세부 하위 비중 용어는 아래를 사용한다.
  - 원화 현금 비중
  - 달러 현금 비중
  - 한국주식 비중
  - 미국주식 비중

## 31. 환율 카드 / 원화·달러 분배 기준 카드 규칙

- 환율 전용 보조 카드는 제거하고, 필요한 정보는 `원화/달러 분배 기준` 카드 안에 통합한다.
- 카드 헤더에는 아래 정보만 둔다.
  - 제목: `원화/달러 분배 기준`
  - 설명: 현재 환율 위치에 따라 국내·해외, 원화·달러 비중을 나눈다는 의미
  - `하나은행 환율 업데이트` 시각
- `네이버 증권` 문구는 노출하지 않는다.
- 현재 환율 숫자 라벨은 흰색 표시바 아래가 아니라 표시바 위에 놓는다.
- 현재 환율 숫자(`1,487원` 같은 값)는 줄바꿈 없이 한 줄로 표시한다.
- 표시바 아래 3개 시나리오 카드는 아래 명칭을 사용한다.
  - 달러 약세 구간
  - 중립 구간
  - 달러 강세 구간

## 32. 국내주식 / 해외자산 데이터 수집 주기와 출처

- 국내주식 실적/점수/추천 스냅샷
  - 파일: `portfolio_dashboard.py`
  - 워크플로: `daily-refresh.yml`
  - 주기: 한국시간 오후 4시 1회
  - 주요 소스:
    - FnGuide 실적/재무/재무비율
    - 네이버 금융 현재가 / 전일종가 / 전일대비
- 국내주식 현재가 빠른 갱신
  - 파일: `portfolio_dashboard.py --price-only`
  - 워크플로: `price-refresh.yml`
  - 주기: 평일 09:10 ~ 15:50, 20분 간격
  - 실적 전망치와 점수는 여기서 갱신하지 않는다.
- 환율
  - 소스: 네이버 증권 모바일 환율 페이지
  - 서버 fetch + 6시간 캐시
  - 실패 시 마지막 성공 스냅샷 fallback
- GLD / 미국주식 Top 10
  - 소스: Stooq
  - 서버 fetch + 6시간 캐시
  - 전일 종가 기반 등락률 계산
  - 실패 시 마지막 성공 스냅샷 fallback
- BTC / ETH
  - 소스: 네이버 모바일 crypto 페이지
  - 서버 fetch + 1시간 캐시
  - 실패 시 마지막 성공 스냅샷 fallback

## 33. 스냅샷 / fallback / GitHub Actions 안정화 규칙

- `dashboard_data.json`에는 국내주식 스냅샷뿐 아니라 환율 및 멀티자산 fallback에 필요한 값도 함께 포함한다.
- `allRankings`에는 성장률 표시용 필드도 함께 저장한다.
- GitHub Actions에서 스냅샷 커밋/푸시가 가능하도록 workflow 권한은 `contents: write` 기준을 사용한다.
- 저장소 Actions 권한도 `Read and write permissions`가 필요하다.
- 데이터 수집 단계는 네트워크 실패를 전제로 재시도 / 백오프 / 안전한 fallback 처리를 우선한다.
- HTML 파싱 시 `.get_text()` 같은 직접 호출은 요소 존재 여부를 먼저 확인해야 한다.
- 개별 종목 실패가 전체 배치를 바로 죽이지 않도록 안전하게 처리하는 것이 운영 기준이다.

## 34. 전일대비 등락률 표시 기준

- 전일대비 등락률은 국내주식만이 아니라 가능한 전 자산에 대해 표시한다.
- 국내주식:
  - 네이버 금융 기준 `전일종가대비등락률` 사용
- 달러:
  - 네이버 환율 페이지의 전일대비 등락률 사용
- GLD / 미국주식:
  - 현재가와 직전 종가를 비교해 등락률 계산
- BTC / ETH:
  - 네이버 crypto 페이지 등락률 사용
- 데이터가 없는 자산은 `0%`로 보이지 않게 하고 `-`로 표시한다.

## 35. ROE 전망치 도입 검토 결과

- 현재 서비스에서 실제 점수 계산에 쓰는 ROE는 FnGuide `SVD_FinanceRatio.asp` 재무비율 테이블의 확정치다.
- 현재 실적 전망치는 FnGuide 메인 재무 테이블에서 읽고 있으므로, 이론상 같은 테이블에서 ROE 전망치도 읽을 수 있을 가능성이 있다.
- 다만 현재 수집 구조에서 안정적으로 쓰는 경로는 재무비율 테이블이며, 이 경로에는 전망 ROE 열이 없다.
- 전망 ROE 도입은 아래 방식이 현실적이다.
  - 전망 ROE가 안정적으로 파싱되면 우선 사용
  - 없으면 기존 확정 ROE fallback
- 현재 기준으로는 아직 전면 도입하지 않고, 확정 ROE 유지가 기본이다.

## 36. 다음 스레드를 위한 아이디어 메모

- 투자성향별 실행 순서를 그대로 적용해 `1일 1회 리밸런싱` 했을 때의 수익률을 계산하고, 실제 내 잔고 수익률과 비교하는 기능
- 국내 상장 미국 지수 추종 ETF도 검색 가능하게 넣되
  - `미국주식` 섹터로 분류
  - 달러와는 무관하게 원화로 매수/매도 처리

## 37. 연도 기준 / ROE / 실적 전망치 해석 규칙 재정의

- `올해` 기준은 종목별 FnGuide 표 헤더가 아니라 실제 달력 연도 기준으로 통일한다.
- 실적 전망치 계산 규칙:
  - 작년 확정 = `현재 연도 - 1`
  - 전망치 우선순위 = `현재 연도 + 2` -> `현재 연도 + 1`
  - 둘 다 없으면 제외
- ROE 계산 규칙:
  - 올해 전망 ROE = `현재 연도`
  - 없으면 작년 확정 ROE = `현재 연도 - 1`
  - 둘 다 없으면 제외
- 더 먼 미래 연도를 임의로 fallback 하는 방식은 사용하지 않는다.
- 연도 숫자는 하드코딩하지 않고, 달력 연도 기준 자동 계산 구조를 유지한다.

## 38. 국내/외부 가격 갱신 워크플로 분리

- 가격 갱신은 아래 2개 액션으로 분리한다.
  - `Domestic Price Refresh`
  - `External Asset Price Refresh`
- 국내 가격 액션:
  - 평일 09:10 ~ 15:50, 20분 간격
  - 국내주식 현재가 전용 갱신
- 외부 자산 액션:
  - 24시간, 20분 간격
  - 환율 / 달러 / GLD / 미국주식 / BTC / ETH 갱신
- 외부 자산은 생략 대상이 아니라 “별도 경로로 계속 갱신”하는 것이 기준이다.
- 두 액션 모두 결과 스냅샷은 `web/public/data/dashboard_data.json` 기준으로 이어받는다.

## 39. 국내주식 price-only 최적화 / fallback 규칙

- 국내 `price-only`는 기존 스냅샷을 재사용하고, 가격 컬럼만 갱신하는 경량 경로를 사용한다.
- 국내 `price-only`에서는 아래를 다시 계산하지 않는다.
  - 시가총액
  - 재무비율
  - 실적 전망치
  - 점수 / 랭킹 / 추천 비중
  - 시장 전체 유니버스 재생성
- 국내 `price-only`에서 갱신 대상은 아래 가격 필드다.
  - 현재가
  - 전일종가
  - 전일종가대비등락률
- 가격 fetch 실패 시에는 종목별로 최근 성공 가격 세트를 유지한다.
  - 현재가
  - 전일종가
  - 전일종가대비등락률
- 국내 `price-only`는 스냅샷 내 종목들을 병렬 fetch 하도록 최적화한다.
- UI에서 가격이 없는 자산은 `0원`으로 보이지 않게 하고 `-`로 표시한다.

## 40. 전일 대비 부호 정규화 / 국내 가격 표시 규칙

- 전일대비 등락률은 파싱된 숫자 부호를 그대로 신뢰하지 않는다.
- `현재가`와 `전일종가`의 실제 관계로 부호를 정규화한다.
  - 현재가 < 전일종가 -> 음수
  - 현재가 > 전일종가 -> 양수
  - 같으면 0
- 이 정규화는 아래 2곳 모두에 적용한다.
  - 실시간 quote 파싱
  - 스냅샷 저장 직전
- 이미 저장된 스냅샷도 같은 규칙으로 다시 정리할 수 있어야 한다.
- 에코프로처럼 `allRankings`에 없더라도 `stockUniverse`에서 가격이 정상 반영되면 보유 자산 표에는 정상 표시되어야 한다.

## 41. 환율 표시값과 분배 계산값 분리

- 환율 분배 계산은 `1300 ~ 1500` 범위에서 clip 하여 사용한다.
- 하지만 UI에 보여주는 현재 환율 숫자는 clip 하지 않고 실제 환율 값을 그대로 보여준다.
- 즉 1,500원을 넘어도 표시 바 위 숫자는 `1,523원`처럼 실제 값이어야 한다.
- 이 분리 원칙은 아래에 적용한다.
  - 원화/달러 분배 기준 카드의 현재 환율 숫자
  - 달러 기준 자산의 원화 환산 표시
  - 거래 입력 시 달러 자산 환산
- 반면 국내/해외, 원화/달러 비중 계산만 clip 로직을 유지한다.

## 최신 통합 기준

이 섹션은 앞선 원본 기록과 후속 반영 사항을 바탕으로, 다음 스레드에서 바로 참고할 수 있도록 현재 기준을 다시 통합 정리한 버전이다.

### 제품 구조

- 서비스명은 `모두의 ETF`
- 서비스는 아래 2개 축으로 구성된다.
  - 추천 포트폴리오 보기
  - 내 포트 관리하기
- 로그인 전에는 추천 포트폴리오 경험 중심
- 로그인 후에는 저장/업데이트/리밸런싱 중심
- 자산배분은 추천 화면의 별도 장식 요소가 아니라, 추천 포트 계산과 현재 내 자산 비교에 동시에 반영되는 핵심 구조이다.

### 로그인 전 경험

- 랜딩 페이지에서는 브랜드와 추천 포트폴리오 진입이 중심이다.
- 메인 브랜드 `모두의 ETF`는 가장 강하게 보여야 한다.
- 메인 문구는 아래를 사용한다.
  - `추천 포트폴리오를 확인하고,`
  - `필요한 포트폴리오 변화를 자동으로 관리하세요.`
- 로그인 전 기본 추천 조건:
  - 종목 카테고리: 기본 관심 종목
  - 투자 성향: 밸런스형
  - 투자 금액: 1,000만원
- 로그인 전에는 추천 포트폴리오와 실행 순서까지만 보여준다.
- 아래 실제 반영 단계는 숨긴다.
  - 현금 조정
  - 장바구니
  - 비교
  - 확정

### 로그인 후 경험

- 로그인 후에는 `내 포트 관리하기`와 `추천 포트폴리오 보기`를 모두 사용할 수 있다.
- 랜딩 상단 CTA는 로그인 후 `로그아웃`, 비로그인 시 `구글로 로그인하기`
- `내 포트 관리하기` 내부에는 별도 로그아웃 버튼을 두지 않는다.
- 처음 로그인한 직후 첫 진입에서는 `내 포트 관리하기`로 이동할 수 있다.
- 다만 그 이후에는 새로고침이나 인증 상태 재확인 시에도 사용자가 마지막에 보던 탭과 스크롤 위치를 유지한다.
- 로그아웃 시에는 새로고침 없이 아래 상태를 즉시 초기화한다.
  - 보유 종목
  - 저장된 포트폴리오 상태
  - 거래/리밸런싱 장바구니
  - 현금 조정
  - 비교/확정 대기 상태

### 추천 점수와 추천 포트폴리오 계산

- 추천 점수는 아래 3팩터를 사용한다.
  - 성장
  - 가치(PER)
  - ROE
- 성향별 가중치는 아래를 사용한다.
  - 안정형: 성장 20 / 가치 50 / ROE 30
  - 밸런스형: 성장 40 / 가치 40 / ROE 20
  - 공격형: 성장 60 / 가치 30 / ROE 10
- 백엔드 기본 프로필 이름은 `균형형`으로 남아 있지만, 실제 가중치 값은 프론트 `밸런스형`과 같은 40 / 40 / 20 기준을 사용한다.
- 추천 포트폴리오는 먼저 상위 자산군 4개를 배분한 뒤, 그중 국내주식 예산만 따로 떼서 생성한다.
  - 현금/예적금/채권
  - 금
  - 가상자산
  - 주식
- 현금과 주식은 환율을 반영하여 아래 하위 축으로 다시 나눈다.
  - 현금: 원화 현금 / 달러 현금
  - 주식: 한국주식 / 미국주식
- 추천 포트폴리오 비중은 점수 제곱 비례로 계산한다.
- 이상적인 주수는 먼저 `floor` 처리하고, 남은 현금은 추천 상위 순서대로 1주씩 반복 배분한다.
- ETF는 선택 대상에서 제외한다.
- 종목 선택은 카테고리 선택 + 개별 검색/추가/제외 방식이다.
- 종목 카테고리 선택 영역은 `국내주식 추천 포트폴리오` 섹션 아래에 둔다.
- 연도 기준은 달력 연도 기준으로 자동 계산한다.
- 실적 전망치:
  - 작년 확정 = `현재 연도 - 1`
  - `현재 연도 + 2` 전망 우선
  - 없으면 `현재 연도 + 1`
  - 그래도 없으면 제외
- ROE:
  - `현재 연도` 전망 우선
  - 없으면 `현재 연도 - 1` 확정
  - 그래도 없으면 제외

### 내 포트 관리하기 흐름

- `내 포트 관리하기`는 아래 순서를 따른다.
  - 현재 보유 포트폴리오 확인
  - 종목 검색 또는 상위 카드에서 거래 입력
  - 거래 장바구니 적재
  - 현금 추가/인출 적재
  - 포트폴리오 업데이트 비교
  - 최종 확정
- 장바구니에 담는 즉시 저장하지 않는다.
- 비교 화면에서 변경 내용을 확인한 뒤 확정할 때만 저장한다.
- 비교 화면은 기존 저장 포트폴리오 vs 업데이트 예정 포트폴리오를 보여준다.
- 상단 요약 카드는 2 x 4 구조를 사용한다.
  - 1행: 총 매입액 / 총 평가액 / 총 평가손익 / 총 실현손익
  - 2행: 원금 / 잔여 현금 / 현재 수익금 / 현재 수익률
- 현재 수익금은 `총 평가손익 + 총 실현손익`
- 현재 수익률은 `원금 대비 현재 수익금 비율`
- 전일대비 등락률은 전체 요약이 아니라 종목별 표 컬럼으로 보여준다.

### 추천 포트폴리오 리밸런싱 흐름

- 리밸런싱도 아래 순서를 따른다.
  - 실행 순서에서 필요한 항목 선택
  - 리밸런싱 장바구니 적재
  - 현금 추가/인출 적재
  - 리밸런싱 전후 포트폴리오 비교
  - 최종 확정
- 리밸런싱 장바구니 아래에서 즉시 반영하지 않는다.
- 확정 버튼은 비교 섹션 아래에 위치한다.

### 현금 조정

- 현금 추가/인출은 `내 포트 관리하기`와 `리밸런싱` 양쪽 모두 지원한다.
- 현금 조정은 참고 정보가 아니라 실제 변경 항목이다.
- 현금 변동은 장바구니에 담아 종목 거래와 함께 한 번에 확정한다.
- 비교 화면에서도 별도 변경 항목으로 보여준다.
- 비교 카드에는 아래 값을 보여준다.
  - 기존 총 원금
  - 변경 후 총 원금
  - 현금 변화

### 장바구니 규칙

- 장바구니는 가능한 한 한 줄형 구조로 정리한다.
- 항목 구성:
  - 종목/설명
  - 거래 주수
  - 거래 단가
  - 예상 금액
  - 제거
- 용어:
  - 매수: `매입 단가`
  - 매도: `매도 단가`
- 매수/매도는 색과 보조 텍스트로 구분한다.
- 장바구니에서 수량/단가 수정 가능
- 장바구니 제거 시 비교 내역에도 즉시 반영

### 색상 규칙

- 이번 프로젝트에서 색의 의미는 아래처럼 통일한다.
  - 양수 / 매수 / 신규 편입 / 비중 확대 = 빨강
  - 음수 / 매도 / 비중 축소 / 전량 매도 = 파랑
- 이 규칙은 아래 전반에 적용한다.
  - 실행 순서
  - 장바구니
  - 변화 배지
  - 업데이트 비교
  - 리밸런싱 비교
  - 제거/삭제 버튼과 위험 계열 UI
- 라이트/다크 모드 모두 같은 의미 체계를 유지해야 한다.

### 비교 화면 규칙

- 비교 화면은 단순 시각화가 아니라 “검토 후 확정” 단계이다.
- 확정 전에는 예정 상태만 보여주고, 확정 후에 실제 저장 결과를 반영한다.
- 도넛 차트 옆 범례는 항상 비중 높은 순으로 정렬한다.
- 비교 카드 목록도 가능하면 반영 후 비중 높은 순으로 정렬한다.
- 변화 카드 구조는 세로형보다 한 줄형이 우선이다.
- 한 줄형 비교 카드는 아래 구성으로 정리한다.
  - 항목명
  - 기존 값
  - 변경 후 값
  - 변화 값
- 현금 변화도 같은 패턴으로 표현한다.

### 현재 보유 포트폴리오 / 실행 순서 UI 원칙

- 현재 보유 포트폴리오:
  - 증권사 잔고 화면처럼 읽기 쉬워야 한다.
  - 각 종목 카드 간격은 적당히 분리되어야 한다.
  - 카드 하단 메타 텍스트가 잘리지 않게 하단 여유가 필요하다.
  - 너무 높지도, 너무 붙지도 않게 미세 조정한다.
  - 종목별 표에는 `전일 대비` 컬럼이 있어야 하며, 데이터가 없으면 `0%` 대신 `-`로 표시한다.
  - `4개 자산군 비중` 도넛 중심에는 종목 수가 아니라 총 평가액을 표시한다.
  - `4개 자산군 비중` 상세 범례는 비중과 평가액을 함께 표시한다.
- 실행 순서:
  - 카드 안에 주수 변화/비중 변화/현재/목표/현재가 정보를 보여준다.
  - 하단 3개 정보 박스는 과도하게 높지 않게 조정한다.
  - 내부 박스 높이를 줄였다면 바깥 카드 높이도 같이 줄여 균형을 맞춘다.
  - 실행순서는 국내주식 중심 흐름을 유지하고, 추천 포트 목표 주수는 정수 주수 기준으로 해석 가능해야 한다.

### 환율 / 자산배분 가이드 표시 규칙

- 환율 설명은 별도 정보 카드보다 `원화/달러 분배 기준` 카드 안에 통합한다.
- 카드에는 아래 정보를 보여준다.
  - 현재 환율 위치
  - 하나은행 환율 업데이트 시각
  - 1,300 / 1,400 / 1,500 구간 기준
  - 달러 약세 / 중립 / 달러 강세 구간별 분배 규칙
- 현재 환율 숫자 라벨은 흰색 표시바 위쪽에 두고 줄바꿈 없이 한 줄로 표시한다.
- `네이버 증권` 브랜드 텍스트는 별도로 강조하지 않는다.
- 환율 비중 계산은 `1300 ~ 1500` 범위에서 clip 하되, 화면에 보이는 현재 환율 숫자와 달러 환산값은 실제 환율을 사용한다.

### 저장/인프라

- 인증은 Google OAuth 사용
- 저장소는 Supabase 사용
- 사용자별 `profiles` 테이블과 RLS 정책 사용
- 여러 포트폴리오 슬롯을 지원한다.
  - 신규 포트폴리오 만들기
  - 복사하기
  - 이름 변경하기
  - 삭제하기
- 동일한 이름의 포트폴리오는 만들 수 없다.
- 국내주식 전체 스냅샷은 GitHub Actions 배치로 관리한다.
  - 오후 4시: 실적/점수/추천 전체 갱신
  - 평일 09:10 ~ 15:50: 20분 간격 현재가 전용 갱신
- 외부 자산 가격 갱신은 별도 액션으로 24시간 20분 간격 운용한다.
- 환율 / GLD / 미국주식 / BTC / ETH는 현재 모두 서버 fetch + 20분 캐시 + 마지막 성공 스냅샷 fallback 구조를 사용한다.
- 국내 `price-only`는 기존 스냅샷을 재사용하고 가격 컬럼만 종목별 갱신한다.
- 국내 `price-only`는 랭킹 / 점수 / 실적 / 시총 / 재무비율을 다시 계산하지 않는다.
- 국내 가격 fetch 실패 시에는 해당 종목의 이전 성공 가격 세트를 유지한다.
- 국내 `price-only`는 스냅샷 내 국내 종목들을 병렬로 fetch 한다.

### 차기 자산배분 기능 확장 시 고려사항

- 자산배분 기능은 별도 고립 화면보다, 추천/업데이트/리밸런싱 흐름 안에 녹아드는 것이 바람직하다.
- 자산배분 변경이 생기면 아래가 함께 일관되게 바뀌어야 한다.
  - 추천 포트 비중
  - 실행 순서
  - 장바구니
  - 비교 화면
- 최종 저장 결과
- 즉, 자산배분도 “설정값”이 아니라 “변경 후 검토/확정 흐름의 일부”로 설계하는 것이 좋다.
- 현재 코드 기준 ROE는 이미 `현재 연도 전망 우선 + 전년도 확정 fallback` 구조를 사용한다.
- 향후 추가 검토 대상은 ROE 전망치 도입 여부가 아니라, 이 ROE 기준을 프론트 점수 안내와 어떻게 더 일관되게 맞출지에 가깝다.

### 작업 마무리 운영 규칙

- Safari에서 개발 서버 기준 CSS 반영이 꼬인 적이 있으므로, 마무리는 항상 프로덕션 빌드 기준으로 확인한다.
- 완료 시 검증 순서:
  - `next build`
  - 기존 3000 포트 서버 종료
  - `next start`
  - `http://127.0.0.1:3000` 응답 확인
- 완료 기준은 `코드 수정 완료`가 아니라 `최신 빌드가 3000 포트에서 정상 오픈 확인 완료`이다.

## 빠른 참조용 구조화 요약

이 섹션은 다음 스레드에서 빠르게 맥락을 파악할 수 있도록,
현재 기준을 기능 축이 아니라 `UI / 기능 흐름 / 데이터 계산 / 저장·운영 규칙` 축으로 다시 정리한 것이다.

### UI

#### 랜딩 / 로그인 전 화면

- 브랜드명 `모두의 ETF`가 가장 먼저 강하게 보인다.
- 메인 문구는 아래를 사용한다.
  - `추천 포트폴리오를 확인하고,`
  - `필요한 포트폴리오 변화를 자동으로 관리하세요.`
- 로그인 전에는 추천 포트폴리오 보기 경험만 보여준다.
- 로그인 전 추천 화면에서는 실행 순서까지만 보여주고, 그 아래 실제 반영 단계는 숨긴다.
- 로그인 전 기본 추천 조건은 아래와 같다.
  - 종목 카테고리: 기본 관심 종목
  - 투자 성향: 밸런스형
  - 투자 금액: 1,000만원

#### 로그인 후 화면

- 로그인 후에는 `내 포트 관리하기`와 `추천 포트폴리오 보기`를 모두 사용할 수 있다.
- 상단 CTA는 로그인 후 `로그아웃`, 비로그인 시 `구글로 로그인하기`로 동작한다.
- `내 포트 관리하기` 내부에는 별도 로그아웃 버튼을 두지 않는다.
- 처음 로그인한 직후에는 관리 탭 진입이 가능하지만, 이후에는 사용자가 마지막에 보던 탭과 스크롤 위치를 유지한다.

#### 현재 보유 포트폴리오 UI

- 증권사 잔고 화면처럼 읽기 쉬워야 한다.
- 상단 요약 카드는 2 x 4 구조를 따른다.
  - 1행: 총 매입액 / 총 평가액 / 총 평가손익 / 총 실현손익
  - 2행: 원금 / 잔여 현금 / 현재 수익금 / 현재 수익률
- 종목 카드 안에는 아래 정보가 보인다.
  - 보유 주수
  - 평균 매입가
  - 현재가
  - 매입총액
  - 평가총액
  - 평가손익
  - 수익률
  - 전일 대비
- 카드 하단 메타 정보는 잘리지 않게 충분한 하단 여유가 있어야 한다.
- 카드 간격은 실행순서보다 약간 차분하지만, 너무 붙지 않게 유지한다.
- 4개 자산군 도넛 중심에는 총 평가액을 보여준다.
- 4개 자산군 상세 범례는 비중과 평가액을 함께 보여준다.

#### 실행 순서 UI

- 각 실행 카드는 아래 정보를 함께 보여준다.
  - 종목명
  - 실행 유형
  - 변화 배지
  - 현재 비중 / 변경 후 비중
  - 현재 / 목표 / 현재가
- 하단 3개 정보 박스는 과도하게 높지 않게 조정한다.
- 내부 박스 높이를 줄였다면 바깥 카드 높이도 같이 줄여 균형을 맞춘다.

#### 장바구니 UI

- 장바구니는 한 줄형 구조를 기본으로 한다.
- 공통 항목:
  - 종목/설명
  - 거래 주수
  - 거래 단가
  - 예상 금액
  - 제거
- 매수/매도에 따라 색과 보조 텍스트가 구분된다.
- `제거` 버튼은 라이트/다크 모두에서 또렷하게 보여야 한다.

#### 비교 화면 UI

- 비교 화면은 단순 시각화가 아니라 “검토 후 확정” 단계이다.
- 위에는 도넛 차트와 범례
- 아래에는 변화 카드 목록
- 변화 카드는 세로형보다 한 줄형 구조를 우선한다.
- 한 줄형 비교 카드는 아래 구성을 따른다.
  - 항목명
  - 기존 값
  - 변경 후 값
  - 변화 값
- 현금 변화도 동일한 패턴으로 표현한다.
- 현재 내 자산 비중 상세 설명은 긴 문장 대신 `현재 vs 가이드` 미니 바 차트 + `+/- %p` 차이값 구조를 우선한다.
- 상세 설명 순서는 차이 큰 순이 아니라 왼쪽 범례 순서를 그대로 따른다.

#### 색상 규칙

- 양수 / 매수 / 신규 편입 / 비중 확대 = 빨강
- 음수 / 매도 / 비중 축소 / 전량 매도 = 파랑
- 이 규칙은 아래에 일관되게 적용한다.
  - 실행 순서
  - 장바구니
  - 변화 배지
  - 업데이트 비교
  - 리밸런싱 비교
- 라이트/다크 모드 모두 같은 의미 체계를 유지해야 한다.
- 빠른 단가 조정은 아래 색 구성을 기준으로 사용한다.
  - `단가 내리기` = 붉은 계열
  - `단가 올리기` = 푸른 계열

#### 환율 / 자산배분 가이드 UI

- 자산배분 가이드의 하위 용어는 아래를 사용한다.
  - 원화 현금 비중
  - 달러 현금 비중
  - 한국주식 비중
  - 미국주식 비중
- 환율 설명은 `원화/달러 분배 기준` 카드 안에 통합한다.
- 카드에는 아래 내용을 보여준다.
  - 하나은행 환율 업데이트 시각
  - 현재 환율 위치 바
  - 달러 약세 / 중립 / 달러 강세 구간 설명
- 현재 환율 값은 표시바 위쪽에 한 줄로 표시한다.
- 표시 환율은 실제 환율값을 사용하고, 비중 계산만 clip 값을 사용한다.

### 기능 흐름

#### 로그인 전 흐름

- 랜딩 진입
- 추천 포트폴리오 확인
- 실행 순서 확인
- 로그인 유도

#### 내 포트 관리하기 흐름

- 현재 보유 포트폴리오 확인
- 거래 종목 검색 또는 상위 카드에서 종목 선택
- 거래 주수/단가 입력
- 매수 또는 매도 장바구니 담기
- 필요하면 현금 추가/인출도 장바구니 담기
- 포트폴리오 업데이트 비교 확인
- 최종 확정

#### 리밸런싱 흐름

- 실행 순서에서 필요한 항목 선택
- 리밸런싱 장바구니 담기
- 필요하면 현금 추가/인출 담기
- 리밸런싱 전후 포트폴리오 비교
- 최종 확정

#### 공통 UX 원칙

- 장바구니에 담는 즉시 저장하지 않는다.
- 사용자가 비교 화면에서 변경 내용을 확인한 뒤 확정할 때만 저장한다.
- `장바구니 -> 비교 -> 확정` 구조를 `내 포트 관리하기`와 `리밸런싱` 양쪽에 최대한 통일한다.
- 로그아웃 시에는 새로고침 없이 작업 공간 전체를 기본 상태로 즉시 초기화한다.
- 추천 목표 주수는 먼저 정수 주수로 내린 뒤, 남은 현금은 추천 상위 순서대로 1주씩 반복 배분한다.

### 데이터 계산

#### 추천 점수

- 추천 점수는 성장 / 가치(PER) / ROE 3팩터를 사용한다.
- 가중치는 아래와 같다.
  - 안정형: 성장 20 / 가치 50 / ROE 30
  - 밸런스형: 성장 40 / 가치 40 / ROE 20
  - 공격형: 성장 60 / 가치 30 / ROE 10
- 백엔드 내부 기본 프로필 이름은 `균형형`으로 남아 있지만, 실제 가중치 값은 프론트 `밸런스형`과 같은 구조다.

#### 연도 기준

- 성장/PER 시작점:
  - `기준 연도 - 1` 확정 실적
- 성장/PER 끝점:
  - `기준 연도 + 2` 전망치 우선
  - 없으면 `기준 연도 + 1` 전망치
- ROE:
  - `기준 연도` 전망값 우선
  - 없으면 `기준 연도 - 1` 확정값
- 연도 기준은 종목별 표 헤더가 아니라 달력 연도 기준 자동 계산을 따른다.

#### 제외 종목 로직

- `기준 연도 + 2` 전망치가 있으면 사용
- 없으면 `기준 연도 + 1` 전망치 사용
- 둘 다 없으면 제외
- 금융/증권 업종은 별도 대응

#### 비중 계산

- 추천 포트폴리오 비중은 점수 제곱 비례를 사용한다.
- 도넛 차트 옆 범례는 항상 비중 높은 순으로 정렬한다.
- 비교 카드 목록도 가능하면 반영 후 비중 높은 순으로 정렬한다.
- 목표 주수는 정수로 변환한 뒤 남은 현금을 추천 상위 순서대로 재배분한다.

#### 현금 계산

- 현금 추가/인출은 포트 변경의 일부로 취급한다.
- 비교 화면에서는 별도 항목으로 아래 값을 보여준다.
  - 기존 총 원금
  - 변경 후 총 원금
  - 현금 변화
- 비로그인 추천 계산은 1,000만원 기준이어야 한다.

#### 데이터 수집 주기

- 국내주식 실적/점수/추천 스냅샷은 오후 4시 1회 갱신한다.
- 국내주식 현재가는 평일 09:10 ~ 15:50 사이 20분 간격으로 추가 갱신한다.
- 외부 자산은 별도 액션으로 24시간 20분 간격 갱신한다.
- 환율 / GLD / 미국주식 / BTC / ETH는 현재 서버 fetch + 20분 캐시 구조를 따른다.
- 외부 fetch 실패 시 마지막 성공 스냅샷 fallback을 사용한다.
- 국내 `price-only`는 기존 스냅샷을 재사용하고, 가격 컬럼만 병렬로 갱신한다.
- 국내 `price-only`에서는 랭킹 / 점수 / 실적 / 시총 / 재무비율은 재계산하지 않는다.

### 저장·인증·운영 규칙

#### 인증 / 저장

- 인증은 Google OAuth 사용
- 저장소는 Supabase 사용
- 사용자별 `profiles` 테이블과 RLS 정책 사용
- 여러 포트폴리오 슬롯 지원
  - 신규 포트폴리오 만들기
  - 복사하기
  - 이름 변경하기
  - 삭제하기
- 동일 이름 포트폴리오는 허용하지 않는다.

#### 저장 시점

- 거래를 입력하거나 장바구니에 담는 단계에서는 저장하지 않는다.
- 확정 버튼을 눌렀을 때만 실제 저장한다.
- 저장 후에는 완료 메시지를 보여준다.

#### 로그아웃 처리

- 로그아웃 시 아래를 즉시 초기화한다.
  - 보유 종목
  - 저장 포트 상태
  - 거래/리밸런싱 장바구니
  - 현금 조정
  - 비교/확정 대기 상태

#### 작업 마무리 규칙

- Safari에서 개발 서버 기준 CSS 반영이 꼬인 적이 있으므로,
- 마무리는 항상 프로덕션 빌드 기준으로 확인한다.
- 완료 시 검증 순서:
  - `next build`
  - 기존 3000 포트 서버 종료
  - `next start`
  - `http://127.0.0.1:3000` 응답 확인
- 완료 기준은 `코드 수정 완료`가 아니라 `최신 빌드가 3000 포트에서 정상 오픈 확인 완료`이다.

#### 운영 안정화 규칙

- GitHub Actions는 스냅샷 커밋/푸시를 위해 `contents: write` 권한이 필요하다.
- 데이터 수집은 네트워크 실패를 전제로 재시도 / 백오프 / fallback 구조를 우선한다.
- HTML 파싱은 요소 존재 여부를 확인한 뒤 텍스트를 읽어야 한다.
- 워크플로 커밋 단계에서는 `.gitignore`에 막히는 `outputs/dashboard_data.json`를 add 하지 않고, `web/public/data/` 기준으로만 커밋한다.
- 전일 대비 등락률은 파싱 문자열보다 `현재가/전일종가` 관계를 우선해 부호를 정규화한다.
